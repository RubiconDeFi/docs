## Стратег

Стратеги имеют возможность размещать заявки и/или заявки, используя ликвидность BathToken для целей маркет-мейкинга в книге заказов Rubicon. Они используют офчейн-стратегии для размещения сделок в рамках системы пулов и получения вознаграждений.

### Точки входа и рычаги стратега

У стратега в системе пулов есть три ключевые функции, которые они могут использовать для создания рынка на Рубиконе. Стратег получает выгоду от выигрыша доли всех исполнений, которые он создает во время маркетмейкинга (см. `strategistBootyClaim`) с помощью средств `bathToken`. Три основные функции заключаются в следующем:

* `executeStrategy` - это основная функция, которую стратег использует для размещения сделок в замкнутой системе. Это позволяет размещать бид и/или аск в книге ордеров при регистрации стратега и ключевых данных, чтобы они могли быть вознаграждены за положительное NPV рыночное поведение.
* `bathScrub` - эта функция является обязательной функцией "очистки", которая должна выполняться в бассейнах Rubicon для эффективной работы. Эта функция анализирует невыполненные заказы для данной BathPair. Это гарантирует, что после «истечения времени» любые ордера, все еще находящиеся в книге ордеров, регистрируются для получения доходности, а ликвидность возвращается в пул. Более того, эта функция должна быть вызвана, чтобы уменьшить невыполненное количество ордеров для данной пары, чтобы соответствовать `maxOutstandingPairCount`
* `removeLiquidity` - эта функция позволяет стратегу «убрать риск» и удалить ликвидность из книги ордеров, которую они разместили с ликвидностью пулов.

### Ключевые идеи

У стратегов есть основная точка входа для использования пользовательских средств и маркет-мейка на Rubicon Market: функция «executeStrategy» на BathPair.sol. Роль стратега заключается в том, чтобы находить точные предложения для использования средств пользователей, которые автоматически и исключительно размещаются в книге заказов Рубикона; Пулы позволяют разместить заявку или заявку в книге заказов Рубикона.

Эта функция имеет ряд входных данных, __ которые напрямую отображаются в функцию «offer» на RubiconMarket.sol:

```
function executeStrategy(
        address targetStrategy, //address of the strategy to use - PairsTrade in v0
        uint256 askNumerator, // *Asset Amt* / Quote Amt
        uint256 askDenominator, // Asset Amt / *Quote Amt*
        uint256 bidNumerator, // *Quote Amt* / Asset Amt
        uint256 bidDenominator // Quote Amt / *Asset Amt*
    )
```

* `targetStrategy` - адрес контракта, реализующего маркетмейкерскую стратегию с заданными входами. В текущей версии пулов [_only approved strategy is the Pairs Trade_](/docs/protocol/rubicon-pools/bathpair).
* `askNumerator` - точный числитель ([pay\_am](/docs/protocol/rubicon-market/key-functions#offer)[t](https://www.youtube.com/watch?v=dQw4w9WgXcQ)), который будет использоваться для совершения сделки с использованием средств LP в количестве актива (например, WBTC ).
* `askDenominator` - точный знаменатель ([buy\_amt](/docs/protocol/rubicon-market/key-functions#offer)), который будет использоваться для совершения сделки с фондами LP в количестве котируемого актива (например, USDC).
* `bidNumerator` - точный числитель ([pay\_am](/docs/protocol/rubicon-market/key-functions#offer)[t](https://www.youtube.com/watch?v=dQw4w9WgXcQ)), который будет использоваться для совершения сделки с использованием средств LP в размере котируемого актива (например, USDC).
* `bidDenominator` - точный знаменатель ([buy\_amt](/docs/protocol/rubicon-market/key-functions#offer)), который будет использоваться для этой ставки в сумме актива (например, WETH).

Стратег должен вне сети определить лучшие заявки и запросы на основе рыночных условий для средств пользователей, чтобы получить выплату за успешное исполнение.
