## 策略师

策略师可以使用 BathToken 流动性进行出价和/或要价，以便在 Rubicon 订单簿上进行做市。他们利用链下策略在矿池系统范围内进行交易并赢得奖励。

### 战略家切入点和杠杆

Pools 系统中的策略师具有三个关键功能，他们可以利用这些功能在 Rubicon 上做市。策略师享受在使用“bathToken”资金进行做市时（参见“strategistBootyClaim”）赢得他们创造的所有填充的一部分的好处。三个主要功能如下：

* `executeStrategy` - 这是策略师用来在闭环系统内进行交易的核心功能。它允许在记录策略师和关键数据的同时在订单簿中放置出价和/或要价，以便他们可以因 NPV 积极的做市行为而获得奖励
* `bathScrub` - 此功能是强制性的“清洁”功能，必须在 Rubicon 池上执行才能有效运行。此函数解析给定“BathPair”上的未完成订单。这确保在“时间到期”之后，仍然在订单簿中的任何订单都会被记录以获得收益，并且流动性会返回到池中。此外，必须调用此函数来减少给定对的未完成订单数量以遵守 maxOutstandingPairCount
* `removeLiquidity` - 此功能允许策略师“规避风险”并从他们使用 Pools liquidity 放置的订单簿中删除流动性。

### 关键思想

为了在 Rubicon 市场上利用用户资金和做市商，策略师有一个核心切入点：BathPair.sol 上的“executeStrategy”功能。策略师的作用是找到准确的报价，以使用自动且排他性地放置在 Rubicon 订单簿中的用户资金；池允许在 Rubicon 订单簿中进行出价或要价。

此函数有许多输入 __ 直接映射到 RubiconMarket.sol 上的“offer”函数：

```
function executeStrategy(
        address targetStrategy, //address of the strategy to use - PairsTrade in v0
        uint256 askNumerator, // *Asset Amt* / Quote Amt
        uint256 askDenominator, // Asset Amt / *Quote Amt*
        uint256 bidNumerator, // *Quote Amt* / Asset Amt
        uint256 bidDenominator // Quote Amt / *Asset Amt*
    )
```

* `targetStrategy` - 使用给定输入执行做市策略的合约地址。在当前版本的池中 [_only approved strategy is the Pairs Trade_](/docs/protocol/rubicon-pools/bathpair)。
* `askNumerator` - 确切的分子 ([pay\_am](/docs/protocol/rubicon-market/key-functions#offer)[t](https://www.youtube.com/watch?v=dQw4w9WgXcQ)) 将用于与 LP 基金进行资产数量交易（例如 WBTC ).
* `askDenominator` - 精确的分母 ([buy\_amt](/docs/protocol/rubicon-market/key-functions#offer)) 将用于与 LP 基金以报价资产金额（例如 USDC）进行交易。
* `bidNumerator` - 将用于与 LP 基金以报价资产金额（例如美元）。
* `bidDenominator` - 将用于以资产金额（例如 WETH）进行此出价的确切分母 ([buy\_amt](/docs/protocol/rubicon-market/key-functions#offer))。

策略师应确定链下最佳出价，并要求根据用户资金的市场情况进行出价，以便获得成功填充的支出。
